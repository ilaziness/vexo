# Vexo 数据同步使用指南

本文档介绍如何使用 Vexo 的数据同步功能，在多台设备间同步您的 SSH 配置和书签数据。

## 概述

Vexo 的数据同步功能允许您：
- 将本地数据备份到远程服务器
- 从远程服务器恢复数据到本地
- 保留最多 5 个历史版本
- 在多台设备间保持数据一致

## 安全说明

**重要**: 您的数据安全是我们的首要考虑：
- 所有数据在本地加密后才上传到服务器
- 服务器无法解密您的数据
- 加密密钥从您的用户密钥派生，不会传输到服务器
- 请妥善保管您的同步 ID 和用户密钥

## 配置同步

### 1. 获取同步凭证

您需要拥有 sync-backend 服务端的管理员为您生成同步凭证：

```bash
# 在服务端执行
./sync-backend gen-user
```

您将获得：
- **Sync ID**: 您的唯一标识符
- **User Key**: 用于加密数据的密钥

### 2. 配置客户端

打开 Vexo 应用，进入 **设置 → 数据同步**：

1. **服务器地址**: 输入 sync-backend 服务地址
   - 例如: `http://localhost:8080` 或 `https://sync.example.com`

2. **同步 ID**: 输入管理员提供的 Sync ID

3. **用户密钥**: 输入管理员提供的 User Key

4. 点击 **保存配置**

配置保存后，系统会自动检查服务器连接状态。

## 同步操作

### 上传数据

将本地数据备份到服务器：

1. 进入 **设置 → 数据同步**
2. 点击 **上传数据** 按钮
3. 等待上传完成

**注意事项**:
- 上传前会自动计算数据哈希用于完整性校验
- 每次上传会创建一个新版本
- 最多保留 5 个历史版本，超出会自动删除最旧版本
- 两次上传间隔需至少 1 分钟

### 恢复数据

从服务器恢复数据到本地：

1. 进入 **设置 → 数据同步**
2. 在 **历史版本** 列表中查看可用版本
3. 点击 **恢复数据** 按钮
4. 在弹出的对话框中选择要恢复的版本
5. 点击 **恢复** 确认

**注意事项**:
- 恢复操作会覆盖本地当前数据
- 恢复前会自动备份当前数据（.backup 后缀）
- 恢复后会进行完整性校验

### 查看历史版本

在 **历史版本** 列表中，您可以查看：
- 版本号
- 文件大小
- 创建时间
- 文件哈希（前 8 位）

点击 **刷新版本** 按钮可更新列表。

## 进度和状态

### 同步进度

上传或下载时，界面会显示：
- 当前阶段（打包、加密、上传/下载、解密、解压）
- 进度百分比
- 已传输数据量 / 总数据量
- 传输速度

### 连接状态

配置区域会显示服务器连接状态：
- **已连接**: 可以正常同步
- **未连接**: 请检查服务器地址和网络

### 完整性校验

同步完成后，系统会自动进行完整性校验：
- **校验通过**: 本地数据与服务器一致
- **校验失败**: 数据可能损坏，建议重新同步

## 常见问题

### Q: 忘记了用户密钥怎么办？

**A**: 用户密钥是加密数据的唯一凭证，无法找回。请联系服务端管理员重新生成新的同步凭证。

### Q: 可以在多台设备上使用相同的同步凭证吗？

**A**: 可以。在多台设备上配置相同的同步凭证，即可实现数据同步。注意：同时修改数据可能导致版本冲突。

### Q: 同步失败怎么办？

**A**: 
1. 检查网络连接
2. 确认服务器地址正确
3. 检查同步 ID 和用户密钥是否正确
4. 查看错误提示信息
5. 如果频繁失败，可能是频率限制，请等待 1 分钟后重试

### Q: 数据版本满了怎么办？

**A**: 系统会自动保留最新的 5 个版本，旧版本会自动删除。如需立即清理，请联系服务端管理员。

### Q: 同步过程中可以取消吗？

**A**: 可以。点击 **取消** 按钮即可中断同步操作。已传输的数据不会保留。

## 最佳实践

1. **定期同步**: 建议定期上传数据，保持备份最新
2. **重要操作前备份**: 在进行大量修改前，先上传一次数据
3. **验证恢复**: 首次配置后，建议测试恢复功能确保正常
4. **安全存储凭证**: 将同步 ID 和用户密钥保存在安全的地方
5. **使用 HTTPS**: 生产环境务必使用 HTTPS 连接服务端

## 故障排查

### 无法连接到服务器

- 检查服务器地址是否正确
- 确认网络连接正常
- 检查防火墙设置
- 确认服务端正在运行

### 上传失败

- 检查数据目录是否存在且可访问
- 确认未超过频率限制（每分钟 1 次）
- 查看磁盘空间是否充足

### 恢复失败

- 确认选择的版本存在
- 检查本地磁盘空间是否充足
- 确认有写入数据目录的权限

### 完整性校验失败

- 可能是传输过程中数据损坏
- 建议重新上传或恢复
- 如多次失败，请检查网络稳定性
