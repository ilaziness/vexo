## Why

用户需要在多台设备间同步 Vexo 的 SSH 配置、书签等数据。当前应用数据仅存储在本地，当用户切换设备或需要备份时，数据迁移困难。通过实现用户数据同步功能，用户可以将配置数据安全地同步到云端，并在需要时恢复到任意设备。

## What Changes

- **新增 sync-backend 服务**: 独立的数据同步服务端应用，支持多用户数据存储和版本管理
- **新增客户端同步模块**: 位于 `internal/sync` 包，实现数据流式打包、压缩、加密和与服务端通信
- **新增配置项**: 同步 ID、用户密钥的配置管理
- **新增版本管理**: 服务端保留最多 5 个历史版本，支持版本恢复
- **新增 CLI 工具**: sync-backend 支持生成用户同步 ID 和密钥
- **新增同步频率控制**: 限制用户同步频率，防止滥用

## Capabilities

### New Capabilities

- `sync-client`: 客户端数据同步功能，包括数据流式打包压缩加密、上传下载和版本恢复
- `sync-server`: 服务端数据同步服务，包括用户管理、多版本存储、数据分发和同步频率控制
- `sync-config`: 同步配置管理，包括同步 ID 和用户密钥的配置

### Modified Capabilities

- 无现有功能需要修改

## Impact

- **新增依赖**: 客户端需要 tar/gzip 压缩库、流式加密库；服务端需要 GORM、SQLite 驱动
- **新增目录**: `internal/sync` (客户端同步包), `sync-backend/` (独立服务端应用)
- **配置变更**: ConfigService 新增同步相关配置字段
- **数据库**: sync-backend 使用 SQLite 数据库存储用户信息和版本元数据
- **文件存储**: 同步数据文件保存到文件系统，按用户 ID 分目录存储
- **安全性**: 数据使用从 user_key 派生的加密密钥进行流式加密，边打包边加密，服务端仅存储加密数据
- **频率控制**: 服务端实现同步频率限制（如每分钟最多 1 次）
