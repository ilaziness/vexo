# sync-server

## Purpose

Server-side data sync service, including user management, multi-version storage and data distribution.

## Responsibilities

- User management and authentication (via sync ID and user key)
- Multi-version data storage (retain up to 5 historical versions)
- Data distribution (upload/download endpoints)
- Version management and recovery support
- CLI tool for generating user sync ID and keys

## Dependencies

- GORM ORM
- MySQL or SQLite database driver
- HTTP server framework
- Crypto library for key generation

## Data Storage

- User sync credentials
- Encrypted data blobs (up to 5 versions per user)
- Version metadata (timestamp, size, checksum)

## Related Capabilities

- sync-client: Client that connects to this service
- sync-config: Credentials generated by this service's CLI
